How Real-Time Network Steganography Works:
Sender Process
  1. Encrypt message with AES
  2. Convert to binary data
  3. Send packets with timing delays
  4. 0 = short delay, 1 = long delay
Receiver Process
  1. Monitor network traffic
  2. Capture packet timestamps
  3. Calculate timing intervals
  4. No log file needed!
Extraction Process
  1. Analyze timing patterns
  2. Reconstruct binary data
  3. Decrypt with password
  4. Reveal original message
Capture Instructions:

Using Wireshark:

  1. Start Wireshark capture on your network interface
  2. Apply filter: host google.com and port 80
  3. Sender starts transmission
  4. Stop capture after transmission completes
  5. Save as PCAP file and upload here

Using Command Line:

tshark -i any -f "host google.com and port 80" -a duration:300 -w capture.pcap
