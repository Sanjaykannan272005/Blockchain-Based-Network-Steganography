Feature Overview

What it does: Allows you to activate or deactivate steganography channels using blockchain signals.

How it works: Publish a special keyword to the blockchain → All agents monitoring the blockchain detect it automatically → Channels activate/deactivate → No risky communication needed!

Real Use: "Algorithm compromised! Switch to backup!" → Published to blockchain → All agents see it in next block → Automatic synchronization.

Common Trigger Examples
Algorithm Switches

  • ALGORITHM_V2 - Use BLAKE2

  • ALGORITHM_V4 - Use KMAC

Operation Controls
  • OPERATION_START - Begin operation
  • OPERATION_PAUSE - Pause activity
  • OPERATION_RESUME - Resume activity
  • EMERGENCY_STOP - Immediate shutdown
  • AGENT_COMPROMISE - Agent compromised
